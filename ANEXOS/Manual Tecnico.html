🏗️ Arquitectura de la Aplicación

Patrón MVC (Model-View-Controller)

COA-Report sigue una arquitectura MVC clásica con separación clara de responsabilidades:

🎨 Vista (JavaFX)

Tecnología: JavaFX 17

Archivos: FXML files en src/main/resources

Define la interfaz gráfica usando archivos FXML declarativos.

🎮 Controlador

Ubicación: com.coa.ui.*Controller.java

Lógica de interacción entre vista y modelo, event handlers.

📊 Modelo

Ubicación: com.coa.model.*

POJOs que representan entidades: Usuario, Empresa, Residuo, etc.

💾 Persistencia

Ubicación: com.coa.dao.*DAO.java

Data Access Objects para comunicación con base de datos SQLite.

Flujo de Datos

Usuario (Input)
       ↓
   Controlador (Event Handler)
       ↓
   Servicio/Lógica (Business Logic)
       ↓
   DAO (Data Persistence)
       ↓
   Base de Datos (SQLite)
       ↓
   DAO (Query Results)
       ↓
   Modelo (Data Transfer)
       ↓
   Controlador (Update State)
       ↓
   Vista (JavaFX Refresh)
       ↓
Pantalla (Output)

Descripción de Componentes Clave

📦 Model Package (com.coa.model) +

Contiene las clases POJO que representan entidades del negocio:

  • Usuario: Usuarios del sistema con autenticación
  • Empresa: Empresas (generadoras, transportadoras, destino final)
  • Encargado: Personal responsable de residuos
  • Planta: Plantas pertenecientes a empresas
  • Residuo: Tipos de residuos peligrosos (CRETIB)
  • Manifiesto: Registros de movimiento de residuos

Patrón Soft-Delete: Todos los modelos incluyen campo activo (INTEGER) para mantener histórico.

💾 DAO Package (com.coa.dao) +

Data Access Objects para cada entidad del modelo. Cada DAO implementa:

  • create(T entity) - Insertar nuevo registro
  • read(int id) - Obtener un registro por ID
  • update(T entity) - Actualizar registro existente
  • delete(int id) - Soft-delete (marcar como inactivo)
  • getAll() - Obtener todos (solo activos por defecto)
  • search(String criteria) - Búsqueda avanzada

Importante: SQLite JDBC no soporta RETURN_GENERATED_KEYS. Usamos SELECT last_insert_rowid() para obtener IDs generados.

🎯 Service Package (com.coa.service) +

Servicios de negocio de nivel superior:

  • AuthenticationService: Autenticación y autorización de usuarios
  • ReportService: Generación de reportes en Excel/PDF
  • ImportService: Importación de datos desde archivos
  • BackupService: Copia de seguridad y restauración de BD
🎨 UI Package (com.coa.ui) +

Controladores de JavaFX que manejan la interfaz:

  • LoginController: Pantalla de autenticación
  • MainController: Pantalla principal y navegación
  • *DialogController: Diálogos para CRUD y reportes

Todos los controladores siguen convención: cada FXML tiene un controlador asociado usando fx:controller

🗄️ Base de Datos SQLite

Esquema de Tablas

La base de datos consta de 6 tablas principales relacionadas entre sí:

USUARIOS

Almacena autenticación y autorización de usuarios del sistema.

CREATE TABLE USUARIOS (
    id_usuario       INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre_usuario   TEXT    NOT NULL UNIQUE,
    contrasena_hash  TEXT    NOT NULL,
    rol              TEXT    NOT NULL (ADMIN | USUARIO),
    activo           INTEGER NOT NULL DEFAULT 1,
    fecha_creacion   TEXT    NOT NULL DEFAULT (datetime('now', 'localtime'))
)

Campos:

  • id_usuario: PK - Identificador único del usuario
  • nombre_usuario: Unique - Login único por usuario
  • contrasena_hash: Hash SHA-256 de contraseña
  • rol: ADMIN o USUARIO (control de acceso)
  • activo: Soft-delete (1=activo, 0=inactivo)

EMPRESAS

Catálogo de empresas involucradas en la gestión de residuos.

CREATE TABLE EMPRESAS (
    id_empresa  INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre      TEXT    NOT NULL,
    nra         TEXT    UNIQUE,
    tipo        TEXT    NOT NULL (GENERADORA | TRANSPORTADORA | DESTINO_FINAL),
    contacto    TEXT,
    telefono    TEXT,
    direccion   TEXT,
    activo      INTEGER NOT NULL DEFAULT 1
)

Tipos: GENERADORA (crea residuos), TRANSPORTADORA, DESTINO_FINAL (procesamiento)

PLANTAS

Plantas pertenecientes a cada empresa (relación 1:N con EMPRESAS).

CREATE TABLE PLANTAS (
    id_planta      INTEGER PRIMARY KEY AUTOINCREMENT,
    id_empresa     INTEGER NOT NULL REFERENCES EMPRESAS(id_empresa) ON DELETE CASCADE,
    nombre         TEXT    NOT NULL,
    codigo         TEXT    UNIQUE,
    direccion      TEXT,
    responsable    TEXT,
    telefono       TEXT,
    activo         INTEGER NOT NULL DEFAULT 1
)

RESIDUOS

Catálogo de tipos de residuos con clasificación CRETIB.

CREATE TABLE RESIDUOS (
    id_residuo    INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre        TEXT    NOT NULL,
    clave_cretib  TEXT    NOT NULL,
    unidad_medida TEXT    NOT NULL (KG | LT | TON | M3),
    descripcion   TEXT,
    activo        INTEGER NOT NULL DEFAULT 1
)

CRETIB: Corrosivo, Reactivo, Explosivo, Tóxico, Inflamable, Biológico

ENCARGADOS

Personal responsable de la gestión de residuos por empresa.

CREATE TABLE ENCARGADOS (
    id_encargado    INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre_completo TEXT    NOT NULL,
    cargo           TEXT,
    empresa_id      INTEGER REFERENCES EMPRESAS(id_empresa),
    activo          INTEGER NOT NULL DEFAULT 1
)

MANIFIESTOS (Tabla Central)

Registro detallado de todos los movimientos de residuos peligrosos.

CREATE TABLE MANIFIESTOS (
    id_manifiesto              INTEGER PRIMARY KEY AUTOINCREMENT,
    num_manifiesto             TEXT    NOT NULL UNIQUE,
    id_empresa_generadora      INTEGER NOT NULL REFERENCES EMPRESAS(id_empresa),
    id_residuo                 INTEGER NOT NULL REFERENCES RESIDUOS(id_residuo),
    cretib                     TEXT    NOT NULL,
    cantidad                   REAL    NOT NULL CHECK (cantidad > 0),
    unidad                     TEXT    NOT NULL,
    fecha_recoleccion          TEXT    NOT NULL,
    fecha_acopio               TEXT,
    nra                        TEXT,
    placas_vehiculo            TEXT,
    id_empresa_transportadora  INTEGER REFERENCES EMPRESAS(id_empresa),
    autorizacion_transportista TEXT,
    destino_final              TEXT    NOT NULL,
    autorizacion_destino_final TEXT,
    id_encargado               INTEGER REFERENCES ENCARGADOS(id_encargado),
    observaciones              TEXT,
    fecha_registro             TEXT    NOT NULL DEFAULT (datetime('now', 'localtime')),
    id_usuario_registro        INTEGER REFERENCES USUARIOS(id_usuario)
)

Índices para Optimización

CREATE INDEX idx_manifest_fecha    ON MANIFIESTOS(fecha_recoleccion);
CREATE INDEX idx_manifest_empresa  ON MANIFIESTOS(id_empresa_generadora);
CREATE INDEX idx_manifest_residuo  ON MANIFIESTOS(id_residuo);
CREATE INDEX idx_manifest_num      ON MANIFIESTOS(num_manifiesto);
CREATE INDEX idx_empresa_tipo      ON EMPRESAS(tipo);
CREATE INDEX idx_residuo_activo    ON RESIDUOS(activo);
CREATE INDEX idx_usuarios_activo   ON USUARIOS(activo);
CREATE INDEX idx_plantas_empresa   ON PLANTAS(id_empresa);

Patrones y Consideraciones

  • Foreign Keys: Habilitadas con PRAGMA foreign_keys = ON
  • Soft-Delete: Campo activo mantiene histórico
  • Timestamps: Automáticos con datetime('now', 'localtime')
  • Constraints: Validaciones a nivel BD (CHECK, NOT NULL, UNIQUE)
  • Cascada: Eliminación en cascada para plantas cuando se elimina empresa
ℹ️ Inicialización: El script schema.sql se ejecuta automáticamente en DatabaseManager.initializeDatabase() cuando se inicia la aplicación.

🔌 API Interna

DatabaseManager

Clase singleton que gestiona la conexión con la base de datos.

public static Connection getConnection() throws Exception
Obtiene la conexión a la base de datos (singleton pattern).
Returns: Connection a la BD SQLite
Throws: Exception si hay problemas de conexión
public static void initializeDatabase() throws Exception
Inicializa la BD ejecutando schema.sql si no existe.
Efecto: Crea todas las tablas, índices y datos iniciales

DAO Methods (Standard CRUD)

Todos los DAOs implementan este patrón:

public T create(T entity) throws Exception
Inserta un nuevo registro en la BD.
Parameter: entity - Objeto del modelo a insertar
Returns: El objeto insertado con ID generado
Nota: Usa SELECT last_insert_rowid() para obtener ID
public T read(int id) throws Exception
Obtiene un registro activo por ID.
Parameter: id - Identificador del registro
Returns: Objeto del modelo o null si no existe
public void update(T entity) throws Exception
Actualiza un registro existente.
Parameter: entity - Objeto con datos actualizados
Requiere: entity.id debe ser válido
public void delete(int id) throws Exception
Elimina (soft-delete) un registro marcándolo como inactivo.
Parameter: id - Identificador a eliminar
Efecto: Establece activo=0, no borra datos
public List<T> getAll() throws Exception
Obtiene todos los registros activos (activo=1).
Returns: List de objetos activos

AuthenticationService

public static Usuario authenticate(String username, String password) throws Exception
Verifica credenciales de usuario.
Parameters:
- username: Nombre de usuario
- password: Contraseña en texto plano
Returns: Usuario si es válido, null si no
Hash: Compara con SHA-256 almacenado
public static String hashPassword(String password)
Genera hash SHA-256 de una contraseña.
Parameter: password - Texto plano
Returns: Hash hexadecimal SHA-256

ReportService

public static void exportToExcel(List<Manifiesto> manifiestos, String filePath) throws Exception
Exporta manifiestos a archivo Excel (.xlsx).
Parameters:
- manifiestos: Lista de registros a exportar
- filePath: Ruta del archivo a generar
Tecnología: Apache POI
public static byte[] generatePDF(List<Manifiesto> manifiestos) throws Exception
Genera reporte en PDF para descarga.
Parameter: manifiestos - Datos para el reporte
Returns: Array de bytes del PDF
Formato: Tabla con encabezados y paginación

package com.coa.dao;

import com.coa.model.Transporte;
import com.coa.util.DatabaseManager;
import java.sql.*;
import java.util.*;

public class TransporteDAO {
    private static final String TABLE = "TRANSPORTES";

    public Transporte create(Transporte t) throws Exception {
        String sql = "INSERT INTO " + TABLE + 
                     " (placa, marca, modelo, capacidad_lt, empresa_id, activo) " +
                     "VALUES (?, ?, ?, ?, ?, 1)";
        
        try (Connection conn = DatabaseManager.getConnection();
             PreparedStatement pstmt = conn.prepareStatement(sql)) {
            
            pstmt.setString(1, t.getPlaca());
            pstmt.setString(2, t.getMarca());
            pstmt.setString(3, t.getModelo());
            pstmt.setInt(4, t.getCapacidadLT());
            pstmt.setInt(5, t.getEmpresaId());
            
            pstmt.executeUpdate();
            
            // Obtener ID generado
            try (Statement stmt = conn.createStatement();
                 ResultSet rs = stmt.executeQuery("SELECT last_insert_rowid() as id")) {
                if (rs.next()) {
                    t.setId(rs.getInt("id"));
                }
            }
        }
        return t;
    }

    public List<Transporte> getAll() throws Exception {
        String sql = "SELECT * FROM " + TABLE + " WHERE activo = 1";
        List<Transporte> list = new ArrayList<>();
        
        try (Connection conn = DatabaseManager.getConnection();
             Statement stmt = conn.createStatement();
             ResultSet rs = stmt.executeQuery(sql)) {
            
            while (rs.next()) {
                Transporte t = new Transporte();
                t.setId(rs.getInt("id_transporte"));
                t.setPlaca(rs.getString("placa"));
                // ... mapear otros campos
                list.add(t);
            }
        }
        return list;
    }
    
    // ... implementar otros métodos CRUD
}

// En ReportService.java
public static byte[] generateManifiestosPorEmpresa(int empresaId, 
                                                    LocalDate desde, 
                                                    LocalDate hasta) throws Exception {
    ManifiestoDAO dao = new ManifiestoDAO();
    List<Manifiesto> manifiestos = dao.searchByEmpresaAndDateRange(
        empresaId, desde, hasta
    );
    
    // Crear documento
    XSSFWorkbook wb = new XSSFWorkbook();
    XSSFSheet sheet = wb.createSheet("Manifiestos");
    
    // Agregar datos
    int rowNum = 0;
    for (Manifiesto m : manifiestos) {
        Row row = sheet.createRow(rowNum++);
        row.createCell(0).setCellValue(m.getNumManifiesto());
        row.createCell(1).setCellValue(m.getCantidad());
        // ... más columnas
    }
    
    ByteArrayOutputStream baos = new ByteArrayOutputStream();
    wb.write(baos);
    return baos.toByteArray();
}

🔍 Debugging y Troubleshooting Técnico

Errores Comunes de Compilación

Error: "No compiler is provided"

Causa: JRE en lugar de JDK

java -version  # Debe mostrar "javac"
javac -version # Debe funcionar

# Solución: Instalar JDK (no JRE)
export JAVA_HOME=/usr/libexec/java_home -v 17

Error: "Failed to load class "org.apache.maven.project.ProjectBuilder"

Causa: Repositorio Maven corrupto

# Limpiar caché local
rm -rf ~/.m2/repository/

# Reinstalar dependencias
mvn clean install

Error: "Cannot find symbol"

Causa: Dependencia faltante

# Limpiar e reinstalar
mvn clean compile

# Si persiste, verificar pom.xml y actualizar dependencias
mvn dependency:tree

Errores en Tiempo de Ejecución

Error: "database is locked"

Causa: Múltiples conexiones simultáneas

// En DatabaseManager.java - usar connection pooling
// O aumentar timeout SQLite
PRAGMA busy_timeout = 5000;

Error: "Duplicate column name"

Causa: Schema.sql ejecutado múltiples veces sin IF NOT EXISTS

// Solución: Borrar BD y reiniciar
rm coareport.db
./run-coa.sh  # Se creará nueva BD

Error: "No suitable driver found"

Causa: Falta JDBC driver para SQLite

// Verificar en pom.xml que existe:
<dependency>
    <groupId>org.xerial</groupId>
    <artifactId>sqlite-jdbc</artifactId>
    <version>3.44.0.0</version>
</dependency>

// Reinstalar
mvn clean install

Errores de UI/JavaFX

Error: "Exception in Application.launch()"

Causa: Problema en stage o setup inicial

// Verificar en COAReportApplication.java
try {
    FXMLLoader loader = new FXMLLoader(
        getClass().getResource("/fxml/main.fxml")
    );
    // ...
} catch (Exception e) {
    e.printStackTrace();  // Ver detalles del error
}

Error: "Cannot load FXML"

Causa: Archivo FXML no encontrado

// Verificar estructura de carpetas
src/main/resources/fxml/  # Debe existir

// Verify ruta en código
getClass().getResource("/fxml/main.fxml")  # Debe ser correcto
